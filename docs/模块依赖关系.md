## DDD 多模块项目最终依赖关系图（清晰版）

### 一、模块层级图（从底到顶）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                             第5层：启动层                                    │
│                            ddd-bootstrap                                    │
│                      (SpringBoot 启动类、配置文件)                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                      ↑ 依赖
┌─────────────────────────────────────────────────────────────────────────────┐
│                             第4层：接口层                                    │
│                            ddd-interfaces                                   │
│                    (Controller、Swagger、参数校验)                           │
└─────────────────────────────────────────────────────────────────────────────┘
                                      ↑ 依赖
            ┌─────────────────────────┼─────────────────────────┐
            ↑                         ↑                         ↑
┌───────────────────┐   ┌───────────────────┐   ┌───────────────────┐
│   第3层：应用层    │   │  第3层：基础设施层 │   │   第2层：公共模块   │
│  ddd-application  │   │ ddd-infrastructure │   │    ddd-common     │
│                   │   │                   │   │                   │
│ • 应用服务        │   │ • 仓储实现         │   │ • 常量            │
│ • 命令/查询       │   │ • MyBatis/Redis   │   │ • 枚举            │
│ • DTO             │   │ • JWT/Security    │   │ • 异常            │
│ • 事务管理        │   │ • CQRS实现        │   │ • 统一返回结果     │
└─────────┬─────────┘   └─────────┬─────────┘   └───────────────────┘
          ↑                       ↑
          └───────────┬───────────┘
                      ↑
┌─────────────────────────────────────────────────────────────────────────────┐
│                             第1层：领域层                                    │
│                             ddd-domain                                      │
│                                                                             │
│  • 实体 (Entity)  • 值对象 (Value Object)                                   │
│  • 聚合根 (Aggregate)  • 领域服务接口                                        │
│  • 仓储接口 (Repository)  • 领域事件 (Domain Event)                         │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 二、依赖关系速查表（一句话总结）

| 模块 | 依赖哪些模块 | 不依赖哪些模块 |
|------|-------------|---------------|
| **ddd-common** | 无 | - |
| **ddd-domain** | ddd-common | application, infrastructure |
| **ddd-application** | ddd-common, ddd-domain | infrastructure |
| **ddd-infrastructure** | ddd-common, ddd-domain | application |
| **ddd-interfaces** | ddd-common, ddd-application, ddd-infrastructure | - |
| **ddd-bootstrap** | ddd-interfaces | - |

### 三、依赖关系矩阵

| 模块 ↓ | common | domain | application | infrastructure | interfaces | bootstrap |
|--------|--------|--------|-------------|----------------|------------|-----------|
| common | - | ❌ | ❌ | ❌ | ❌ | ❌ |
| domain | ✅ | - | ❌ | ❌ | ❌ | ❌ |
| application | ✅ | ✅ | - | ❌ | ❌ | ❌ |
| infrastructure | ✅ | ✅ | ❌ | - | ❌ | ❌ |
| interfaces | ✅ | ❌ | ✅ | ✅ | - | ❌ |
| bootstrap | ❌ | ❌ | ❌ | ❌ | ✅ | - |

### 四、依赖流向简图

```
                    ddd-bootstrap
                          ↑
                    ddd-interfaces
                    ↗           ↖
                  ↗               ↖
    ddd-application               ddd-infrastructure
                  ↖               ↗
                    ↘           ↗
                    ddd-domain
                          ↑
                    ddd-common
```

### 五、禁止的依赖（绝对不要做）

```
❌ ddd-application → ddd-infrastructure  (会造成循环)
❌ ddd-infrastructure → ddd-application  (会造成循环)
❌ ddd-domain → ddd-application         (领域层不应依赖应用层)
❌ ddd-domain → ddd-infrastructure      (领域层不应依赖基础设施)
❌ ddd-interfaces → ddd-domain          (应通过 application 间接访问)
```

### 六、各模块详细清单

```yaml
ddd-common (公共模块):
  位置: 最底层
  依赖: []
  被依赖: [domain, application, infrastructure, interfaces]
  包含:
    - 常量类 (Constants)
    - 枚举类 (ErrorCode)
    - 异常类 (BizException)
    - 统一返回结果 (Result)

ddd-domain (领域层):
  位置: 第1层
  依赖: [common]
  被依赖: [application, infrastructure]
  包含:
    - 实体 (User, Role)
    - 值对象 (Email, PhoneNumber, UserId)
    - 聚合根 (UserAggregate)
    - 领域服务接口 (UserDomainService)
    - 仓储接口 (UserRepository)
    - 领域事件 (UserCreatedEvent, UserActivatedEvent)

ddd-application (应用层):
  位置: 第2层
  依赖: [common, domain]
  被依赖: [interfaces]
  包含:
    - 应用服务 (UserApplicationService, AuthApplicationService)
    - 命令/查询接口 (Command, Query)
    - CQRS 总线接口 (CommandBus, QueryBus)
    - DTO (UserDTO, LoginResponseDTO)
    - 命令/查询处理器接口 (CommandHandler, QueryHandler)

ddd-infrastructure (基础设施层):
  位置: 第2层
  依赖: [common, domain]
  被依赖: [interfaces]
  包含:
    - 仓储实现 (UserRepositoryImpl)
    - MyBatis Plus 配置 (MybatisPlusConfig)
    - Redis 服务 (RedisService)
    - JWT 工具 (JwtUtils)
    - Security 配置 (SecurityConfig)
    - Token 服务 (TokenService)

ddd-interfaces (接口层):
  位置: 第3层
  依赖: [common, application, infrastructure]
  被依赖: [bootstrap]
  包含:
    - Controller (UserController, AuthController)
    - CQRS 总线实现 (SimpleCommandBus, SimpleQueryBus)
    - 全局异常处理 (GlobalExceptionHandler)
    - Swagger 配置

ddd-bootstrap (启动层):
  位置: 第4层
  依赖: [interfaces]
  被依赖: []
  包含:
    - SpringBoot 启动类 (BootstrapApplication)
    - 配置文件 (application.yml)
    - 日志配置 (logback-spring.xml)
```

### 七、核心规则（牢记）

```
1. 领域层 = 心脏，不依赖任何技术框架
2. 基础设施 = 手脚，实现领域层定义的接口
3. 应用层 = 大脑，编排领域服务，不关心技术细节
4. 接口层 = 嘴巴，处理外部请求
5. 启动层 = 身体，把一切组装起来

依赖方向永远向内：bootstrap → interfaces → application/infrastructure → domain → common
```

### 八、记忆口诀

```
common 垫底谁都不靠
domain 只认 common 这个宝
application 和 infrastructure 都找 domain 要
interfaces 把两边都找
bootstrap 最后来启动
application 和 infrastructure 千万别互相找
```
